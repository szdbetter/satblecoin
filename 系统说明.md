# 系统说明

## 1. 需求确认（2026-03-04）
- 交付形式：网页仪表盘，主流框架，后端 Python
- 数据范围：USDT/USDC；来源包括 Aave、Morpho、Kamino、MarginFi、DeFiLlama 精选
- DeFiLlama 精选规则：TVL >= 10,000,000 USD
- 刷新机制：页面打开刷新、手动刷新、自动刷新

## 2. 技术架构
- 后端：FastAPI + requests
- 前端：Streamlit + st-aggrid + plotly
- 缓存：后端内存缓存 300 秒
- 自动刷新：前端每 60 秒触发

## 3. 数据来源与真实性策略
- 主接口：`https://yields.llama.fi/pools`
- 字段：`apy/apyBase/apyBaseBorrow/tvlUsd/symbol/project/chain/pool`
- 链接策略：
  - Aave 使用协议 reserve deep link 模板
  - Morpho/Kamino/MarginFi 优先协议入口参数化链接
  - 兜底为 DeFiLlama `pool` 页面链接

## 4. 功能实现记录
- 新建 `backend/service.py`：抓取、过滤、聚合、缓存、链接生成
- 新建 `backend/api.py`：`/api/health` 与 `/api/yields`
- 新建 `frontend/app.py`：
  - 表格筛选排序分页
  - 点击链接跳转
  - 图表（收益率、TVL、借贷分布）
  - 1 分钟自动刷新 + 手动强制刷新
- 新建启动脚本：
  - `start_backend.sh`
  - `start_frontend.sh`

## 4.1 中间过程决策记录
- 发现 `yields.llama.fi/pools` 不直接提供协议深链，因此设计“协议可构造链接 + DeFiLlama池页面兜底”。
- Solana 按需求仅纳入 `Kamino` 与 `MarginFi`；其中 MarginFi 若无 USDT/USDC 池则展示占位状态，不伪造数据。
- 刷新策略冲突（“5分钟更新”与“1分钟自动更新”）采用分层：
  - 后端真实数据抓取缓存 5 分钟
  - 前端轮询刷新 1 分钟
  - 手动刷新可绕过缓存强制更新
- 字段口径统一：
  - 存款利率：`apyBase` 优先，缺失时 `apy`
  - 借贷利率：`apyBaseBorrow`
  - 币种：严格仅 `USDT` / `USDC`
  - DeFiLlama 精选：TVL >= 10,000,000 USD

## 4.2 本轮修复记录（2026-03-04）
- 问题1：表格列名为英文、APY未统一百分比展示。
  - 修复：AgGrid 全部关键列改中文标题，APY列统一 `xx.xx%` 格式化显示。
- 问题2：点击链接无法稳定跳转。
  - 修复：将链接渲染器改为按钮，点击时 `window.open` 并 `stopPropagation`，避免被行点击事件吞掉。
- 问题3：Aave APY 与官网主界面存在口径混淆。
  - 根因：DeFiLlama 同时返回 Aave 主市场与子市场（如 `lido-market`、`horizon-market`），同链同币会出现多条APY。
  - 修复：Aave 默认仅保留主市场（`poolMeta` 为空）用于对比，避免与官网主界面错位。
  - 结果：Ethereum 的 Aave `USDC/USDT` 固定为 2 条主市场记录。
